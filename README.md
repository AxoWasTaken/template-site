# Notes
- If you want, add a .css file, make sure to link it using the <meta> tag.
- You can expand this site structure as much as you want, or even shrink it.
- If you need to, you can create a custom workflow to deploy the site.
- I've added a .nojekyll file, it is a flag to tell GitHub to skip the Jekyll part of the build-and-deployment queue. I've never used Jekyll, I'm not even quite sure what it is but if you need to remove that file.
- Check out my repository [https://github.com/AxosUnion/AxosUnion.github.io](https://github.com/AxosUnion/AxosUnion.github.io), it is similar to this but much larger and more advanced, I find it often helps to look at other people's code to understand how it works.
- Always check the documentation! GitHub docs is a great resource to check for tips and tricks. There are also 'awesome lists', for some reason they have had a rise in popularity recently, but they have some great reference guides if you want to check them out.
- Have fun! I enjoy doing this sort of thing, web development and programming is a great thing to get into. Good luck!
